# 前端使用说明

## 上架 NFT 需要填写的信息

### 1. NFT 合约地址
这是你要上架的 NFT 所在的合约地址。

**默认值**: 部署脚本会自动填充 MockERC721 的地址

**示例**: `0x5FbDB2315678afecb367f032d93F642f64180aa3`

**说明**: 
- 如果使用部署脚本创建的测试 NFT，直接使用默认地址即可
- 如果要上架其他 NFT 合约的 NFT，需要替换为对应的合约地址

### 2. Token ID
你要上架的 NFT 的 Token ID（编号）。

**示例**: `1`, `2`, `3` 等

**说明**:
- 部署脚本会自动给部署者账户铸造 Token ID 1-5 的 NFT
- 你只能上架自己拥有的 NFT
- Token ID 必须是数字

### 3. 价格
NFT 的售价，以 MockToken (MTK) 计价。

**示例**: `100` (表示 100 个 MTK)

**说明**:
- 价格单位是 MTK（部署的 MockERC20 代币）
- 买家需要有足够的 MTK 才能购买
- 价格必须大于 0

## 完整示例

假设你要上架 Token ID 为 1 的 NFT，售价 100 MTK：

```
NFT 合约地址: 0x5FbDB2315678afecb367f032d93F642f64180aa3 (使用默认值)
Token ID: 1
价格: 100
```

点击"上架"后，会执行两个交易：
1. 授权 NFT 给市场合约（setApprovalForAll）
2. 上架 NFT（list）

## 购买 NFT

只需要填写 Listing ID（从市场列表中查看），例如 `0`

购买时会执行两个交易：
1. 授权代币给市场合约（approve）
2. 购买 NFT（buyNFT）

## 取消上架

只需要填写 Listing ID，例如 `0`

注意：只能取消自己上架的 NFT

## 测试流程

1. 使用账户 #0 (部署者) 连接钱包
2. 上架 Token ID 1 的 NFT，价格 100 MTK
3. 切换到账户 #1
4. 给账户 #1 铸造一些 MTK（使用 cast 命令或合约调用）
5. 使用账户 #1 购买 NFT
6. 查看市场列表，确认 NFT 已售出
